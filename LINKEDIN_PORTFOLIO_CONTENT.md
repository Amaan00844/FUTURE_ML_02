## Interview Talking Points

### "Tell me about this project"
```
I built a support ticket classification system that solves a real problem many companies face: the overwhelming manual work of categorizing customer support tickets.

The system uses Natural Language Processing and machine learning to automatically classify tickets into 8 categories and assign priority levels - all in under a second.

What makes this particularly valuable is the measurable business impact. The manual process required nearly 2,000 hours of work annually. My automated solution reduces this to about 13 hours, saving roughly $50,000 per year.

From a technical perspective, I evaluated four different algorithms and chose Linear SVM because it achieved the best F1-score of 85.6% while maintaining sub-second inference time. I used TF-IDF for feature extraction with 5,000 optimized features including bigrams.

The system is production-ready and could easily integrate with platforms like Zendesk or Jira.
```

### "What was the biggest challenge?"
```
The biggest challenge was actually the class imbalance in the dataset. Hardware tickets made up 28% of the data while Administrative Rights was only 3.7%. 

I addressed this in several ways:
1. Used stratified train-test split to ensure balanced representation
2. Chose F1-score as my primary metric instead of accuracy
3. Analyzed per-category performance to identify weak spots
4. Considered techniques like SMOTE but found Linear SVM handled imbalance well naturally

The result was that even the smallest category achieved acceptable 70% F1-score, while most categories exceeded 85%.
```
